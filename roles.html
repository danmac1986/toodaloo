HOW IT WORKS

Roles

We don’t necessarily think it is important that we will assign roles for each team member. It is quite possible we might have certain areas of focus for different people, so it may be easier to complete the tasks in the given timeframe. But for most of the project, we will most likely work on each part as a group. We opted for our team to work in all sections of the development process,as we thought that not only would this be a viable choice, but we all gain experience in each of the development stages. The reason for this being that this assignment has more sections related to the group’s project rather than information about ourselves.
Another reason being that the sections on this assignment can’t really be split evenly. Some sections are more extensive than others and no one person really excels in one specific area. This not only allowed for each team member to have input at each stage, but we were also able to gain new skills and utilise our strengths at the development stages we were experienced in. This provided us with versatility in our positions to utilise creativity, flexibility and innovation. Also, it’s probably best if the group as a whole has an input on all areas considering we have to do a presentation about it for assignment 5.
Not only did we find this to be exciting, but we also found that this made the communication between the group more successful. We agreed that our communication would be over Microsoft Teams, catching up twice or more (if required) weekly. We found that all team members had valid input and had great points to make during discussions.
